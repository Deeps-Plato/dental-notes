    - Test: POST /dictate accepts audio data (raw PCM or WAV), returns transcribed text
    - Test: /dictate returns 503 if Whisper is not loaded (e.g., during LLM extraction)
    - Test: /dictate handles empty audio gracefully (returns empty string)
  </behavior>
  <action>
    1. Create src/dental_notes/ui/dictation.py:
       - FastAPI router with POST /dictate endpoint
       - Accepts audio data via request body (raw bytes) or multipart upload
       - Gets whisper_service from app.state
       - If whisper_service is not loaded, return 503 with message "Whisper model not available (GPU in use for extraction)"
       - Convert incoming audio bytes to numpy float32 array (PCM 16-bit, 16kHz mono)
       - Call whisper_service.transcribe(audio_array)
       - Return JSON: {"text": transcribed_text}
       - This is a simplified version of the full recording pipeline -- single-shot transcription of a short audio clip, no VAD, no chunking

    2. Update src/dental_notes/main.py:
       - Import and include dictation router: `from dental_notes.ui.dictation import router as dictation_router`
       - `app.include_router(dictation_router)`

    3. Update src/dental_notes/static/review.js -- add dictation functions:
       - Use MediaRecorder API (browser) to capture mic audio
       - Add startDictation(textareaId) function:
         * Request mic permission: navigator.mediaDevices.getUserMedia({audio: true})
         * Create MediaRecorder with audio/webm codec
         * Show visual feedback on the dictation button (recording state)
         * On stop: collect recorded blob, convert to WAV/PCM, POST to /dictate
         * Insert returned text at cursor position in the target textarea
         * Trigger auto-resize on the textarea
       - Add stopDictation() function:
         * Stop MediaRecorder
         * Remove recording visual feedback
       - Toggle behavior: click once to start, click again to stop (or use mousedown/mouseup for hold-to-dictate)
       - Per user decision: dictation available on any editable field at any stage
       - Per research recommendation: simple "hold to dictate" or "click to toggle" per field

    4. Update src/dental_notes/templates/review.html:
       - Add a small microphone icon button next to each textarea label
       - Button calls startDictation('textarea-id') / stopDictation()
       - Per user discretion: mic button presented as small icon inline with section label

    5. Create tests/test_dictation.py:
       - Test POST /dictate with sample audio bytes returns text
       - Test 503 when whisper not loaded
       - Test empty audio returns empty text
       - Use FakeWhisperService from conftest

    NOTE on browser audio: The MediaRecorder API captures audio in the browser and sends it to the server. The server-side endpoint receives raw audio, converts to numpy, and transcribes via WhisperService. This is different from the recording pipeline (which uses sounddevice on the server side). Dictation captures audio in the browser because the dentist is interacting with the browser UI directly.

    Alternative if MediaRecorder is problematic: Use the same server-side AudioCapture for a short burst (start capture for N seconds, transcribe, return). This would require a route that starts/stops capture. The browser approach is simpler for field-level dictation since the user is already in the browser.
