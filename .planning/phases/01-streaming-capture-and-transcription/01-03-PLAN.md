    1. Create `src/dental_notes/main.py`:
       - `create_app() -> FastAPI`:
         - Use `@asynccontextmanager` lifespan that:
           - Creates Settings()
           - Creates WhisperService(settings) and calls load_model() (lazy CUDA init at startup)
           - Creates SessionManager(settings) and injects the whisper service
           - Stores session_manager on app.state
           - Stores settings on app.state
           - On shutdown: unloads whisper model, stops any active session
         - Mount static files: `app.mount("/static", StaticFiles(directory=...))` pointing to the static/ dir relative to this file
         - Configure Jinja2Templates pointing to the templates/ dir relative to this file
         - Include router from ui.routes
         - CRITICAL: Do NOT set `host="0.0.0.0"` anywhere. The Settings default is "127.0.0.1" -- PRV-01.
       - Module-level `app = create_app()` for uvicorn
       - Add a `if __name__ == "__main__": uvicorn.run(app, host=settings.host, port=settings.port)` block

    2. Create `src/dental_notes/ui/routes.py`:
       - `router = APIRouter()`
       - `GET /` -- Render index.html template. Pass: current session state, list of input devices (from list_input_devices()), current transcript if session active.
       - `GET /devices` -- Return JSON list of input devices `[{"index": int, "name": str}]`
       - `POST /session/start` -- Accept optional `device` form field (int). Call session_manager.start(mic_device=device). Return HTMX partial for session controls (recording state).
       - `POST /session/pause` -- Call session_manager.pause(). Return HTMX partial (paused state).
       - `POST /session/resume` -- Call session_manager.resume(). Return HTMX partial (recording state).
       - `POST /session/stop` -- Call session_manager.stop(). Return HTMX partial (idle state) with transcript file path info.
       - `GET /session/stream` -- SSE endpoint using EventSourceResponse:
         - Async generator that polls session_manager.get_transcript() every 0.5 seconds
         - When new text appears, yield ServerSentEvent with event="transcript" containing an HTML snippet for HTMX to swap: `<span>{new_text}</span>`
         - Also yield ServerSentEvent with event="level" containing the audio level value (for the level indicator)
         - When session ends (is_active() returns False), yield event="session_end" and return
       - `GET /session/status` -- Return JSON `{"state": str, "transcript_length": int, "level": float}` for status polling fallback
       - All POST routes should handle RuntimeError from invalid state transitions and return HTTP 409 Conflict with error message
       - Access session_manager via `request.app.state.session_manager`

    3. Create `tests/test_routes.py`:
       - Use `httpx.AsyncClient` with `ASGITransport(app=app)` for testing (standard FastAPI test pattern)
       - Mock SessionManager to avoid real audio/GPU dependencies
       - Create a `FakeSessionManager` that mimics state transitions and returns canned data
       - Override `app.state.session_manager` with fake in test setup

       - Test: `test_index_returns_200` -- GET / returns 200 with HTML content
       - Test: `test_devices_returns_list` -- GET /devices returns JSON list
       - Test: `test_start_session` -- POST /session/start returns 200 when idle
       - Test: `test_stop_session` -- POST /session/stop returns 200 when recording
       - Test: `test_invalid_transition_returns_409` -- POST /session/pause when idle returns 409
       - Test: `test_sse_stream_yields_transcript` -- GET /session/stream yields SSE events with transcript data
       - Test: `test_status_endpoint` -- GET /session/status returns correct state JSON

       Note: SSE testing can be tricky with httpx. For the SSE test, it's acceptable to test that the endpoint returns response_class EventSourceResponse and the generator yields expected data by calling the generator function directly rather than through HTTP.
