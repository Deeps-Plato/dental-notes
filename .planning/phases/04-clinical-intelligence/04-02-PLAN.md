<objective>
Add appointment-type templates with prompt composition (CLI-06) and patient summary generation (REV-04) at the backend/model level. Includes auto-detection of appointment type from transcript when no template is explicitly selected.

Purpose: Template-specific extraction produces more focused SOAP notes (e.g., restorative templates emphasize materials/shade/anesthetic, hygiene templates emphasize probing/BOP/home care). When the user leaves the dropdown on "General", the LLM auto-detects the appointment type from the transcript before extraction. Patient summary provides a plain-language handout for patients at ~6th-grade reading level, generated as a second LLM call during the existing GPU handoff window.

Output: AppointmentType enum, PatientSummary model, template prompt overlays, auto-detect classification prompt, composed extraction pipeline with auto-detection fallback, patient summary generation in GPU handoff, updated session persistence with new fields.
</objective>
